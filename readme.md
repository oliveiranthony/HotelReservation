# 🏨 Hotel Reservation

Projeto desenvolvido com foco em prática de **Git** e evolução de código ao longo do tempo.

A ideia principal foi implementar um sistema de reservas de hotel em **Java**, criando **3 versões diferentes da solução**:

- ❌ Uma solução **muito ruim**
- ⚠️ Uma solução **ruim (melhorada)**
- ✅ Uma solução **boa (ideal)**

Todas as versões foram **commitadas separadamente**, permitindo visualizar claramente a evolução do código.

---

## 📚 Objetivo

Este projeto não é apenas sobre resolver o problema, mas sim sobre:

- Praticar **versionamento com Git**
- Demonstrar **evolução de código**
- Entender **más práticas vs boas práticas**
- Melhorar **organização, legibilidade e design**

---

## 🧠 Conceitos aplicados

- Programação Orientada a Objetos (POO)
- Tratamento de exceções
- Manipulação de datas
- Boas práticas de código
- Refatoração
- Organização em camadas (na versão ideal)

---

## 📈 Evolução das soluções

### ❌ Versão 1 - Muito ruim
- Código pouco organizado
- Baixa legibilidade
- Pouca separação de responsabilidades
- Difícil manutenção

### ⚠️ Versão 2 - Ruim
- Algumas melhorias na estrutura
- Código mais legível
- Ainda com problemas de design
- Começo da separação de responsabilidades

### ✅ Versão 3 - Ideal
- Código limpo e organizado
- Uso correto de POO
- Separação de responsabilidades
- Tratamento adequado de exceções
- Melhor manutenção e escalabilidade
